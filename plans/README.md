## Repository conventions executors must retain

- The application now uses Phoenix 1.8.9 and LiveView 1.2.7. Preserve `<Layouts.app flash={@flash}>`, existing hooks, and the product UI when touching LiveViews.
- Preserve the custom client-IP and rate-limit capabilities while hardening them; do not treat them as generated code to remove.
- DNS clustering is intentionally disabled; do not add a `DNSCluster` child.
- Elixir/Phoenix code is formatted with `mix format`; do not hand-format around it.
- Database tests use `RequestBin.DataCase`; endpoint/LiveView tests use `RequestBinWeb.ConnCase`.
- Persistence goes through Ecto/`RequestBin.Repo`; scheduled work uses Oban and its testing helpers.
- Public errors must be stable and must not expose payloads, changesets, stack traces, environment values, or credentials.
- Recent commits generally use conventional prefixes (`fix:`, `feat:`, `test:`, `chore:`). Each plan names an appropriate branch and commit message.
- Do not push, merge, open a PR, or publish an issue unless the operator explicitly asks.

## Findings considered and rejected

- **Original HTTP dependency patch plan** — rejected as executable work because commit `024bc84` independently resolved all four vulnerable version ranges; Plan 001 is retained as an audit record.
- **Generated development/test signing material flagged by secret scanning** — rejected as a production credential finding because the locations are framework-generated non-production configuration and production requires an environment-provided secret. No values are reproduced in these plans.
- **Unauthenticated collection and CSRF-free collector routes** — rejected as a bug because “no registration” and inbound webhook collection are explicit product behavior. Separate read/write credentials remain a direction option.
- **Postgrex notification-channel advisory** — rejected for this batch because the application has no path that feeds request-controlled channel names to `Postgrex.Notifications`; Oban uses internal fixed notification behavior.
- **Mint outbound-client advisories** — rejected as unreachable in current product flows because no request-controlled outbound HTTP operation exists. Reassess if replay/webhook forwarding is added.
- **Immediate Phoenix 1.8/LiveView major migration** — obsolete because commit `024bc84` independently completed the Phoenix 1.8 / LiveView 1.2 upgrade. No separate migration plan remains.
- **Broad context/repository-wrapper refactor** — rejected as low leverage for this small codebase. Correctness, security, and verification work should land first.
